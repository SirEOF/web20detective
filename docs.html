ABOUT
There're currently 2 modules in the package: Arguments Finder (argsfind.py) and Software Detector (softdetect.py).
Arguments Finder can be used to fuzz an application in order to find its arguments.
Software Detector reveals an information from the server response and does several tests for configurational vulnerabilities, dead code and information disclosure flaws.
Backup fuzzer searches for temporary copies of the file

USAGE
module.py --param1 value1 --param2 value2
The main class of this software is common.py. It contains, for example, a makereq() method for sending requests with urllib.
There're parameters, that are common for all modules.
allowed_params = [ 'ajax', 'cut', 'sleep' ]
If you'll type --ajax 1, the requests will be sent with the header X-Requested-With = XMLHttpRequest.
You can type --sleep N, where N is the number of seconds to sleep before each request.
You can specify the --cut regexp, and the content on the received page will be cut according to your regexp.

Usage of Arguments Finder
argsfind.py --url http://example.com [--mode g|p|c --fill 'foo' --base '/path/to/base']
--url is the only required argument. It defines the target URL.
--mode default value is 'g'. It stands for GET method, p stands for POST, c -- for COOKIE.
--fill default value is '1'. It defines, which value will be put into parameters.
--base default value is bases/argsbase.txt. It defines a path to the dictionary with parameter names.

Usage of Software Detector
softdetect.py --url http://example.com [--args foo,bar]
--url is the only required argument. It defines the target URL.
--args argument is required only for the one PHP dead code test. Script will ask you for this parameter, if it is needed. You should specify the list of parameters, that the target has.

Usage of Backup Fuzzer
fuzzbackup.py --url http://example.com/script.php
MISC
There's the args_grabber.php script in misc directory.
Its purpose is to parse PHP source codes and collect the parameter names for the dictionary there.
An output is appended to bases/argsbase.txt
The usage is simple: args_grabber.php /path/to/php/sources
In config.py you can specify additional headers, that would be sent in each request.
